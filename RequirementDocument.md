# 项目管理与过程改进课程网站需求文档（基于项目计划）

## 1. 项目概述
| 维度 | 说明 |
| --- | --- |
| 项目名称 | 项目管理与过程改进课程教学管理平台 |
| 服务对象 | 上海理工大学《项目管理与过程改进》课程的教师、学生 |
| 目标 | 覆盖课程发布、作业与出勤、讨论与互动、统计与演示 |
| 技术栈 | 前端：Vue 3 + Vite + Element Plus；后端：Spring Boot 3.5.6 + MyBatis Plus；数据库：MySQL 8.0|

## 2. 业务目标与成功指标
1. **课堂作业效率**：学生提交作业的线上化率 ≥ 95%，教师批改平均耗时降低 30%。
2. **出勤与互动**：签到全流程线上化，支持实时统计并输出导出报表；讨论区日访问 UV ≥ 70% 的学生。
3. **演示与交付**：在迭代 1 结束时可以完成核心流程演示（登录-作业-出勤）；迭代 2 结束时具备完整后台与讨论区能力，可交付 PPT/演示视频。

## 3. 用户角色与核心痛点
| 角色 | 痛点 | 核心需求 |
| --- | --- | --- |
| 学生 | 信息分散、作业/考勤提醒不及时 | 单一入口完成登录、课程浏览、作业提交、签到、讨论、成绩查询 |
| 教师 | 作业收集零散、批改麻烦、考勤统计耗时 | 快速发布作业、在线批改与反馈、考勤管理与异常处理、统计分析 |
| 助教/管理员 | 需要协助老师维护课程与学生 | 用户/班级维护、权限配置、全局通知 |

## 4. 迭代计划与范围
### 4.1 迭代 1（Day 1~14）——最小可运行版本
- **目标**：前后端打通，完成学生端登录、课程展示、作业提交流程，支持基础考勤。
- **范围**：
  - 认证与基础权限（学生/教师）。
  - 学生首页、课程列表。
  - 作业列表、详情、提交（含文件上传）。
  - 考勤签到/查看。
  - 数据库结构、基础文档、Swagger/API 文档。
- **交付物**：可运行前端、可运行后端、SQL、API 文档（包括 ApiFox/Swagger）、≥10 条测试用例、迭代周报。

### 4.2 迭代 2（Day 15~28）——完整系统
- **目标**：上线教师后台、讨论区、公告系统，形成可完整展示的系统。
- **范围**：
  - 教师批改、考勤管理、公告与讨论区 CRUD。
  - 评论、通知、权限拦截扩展。
  - 系统 UI 优化、PPT、测试报告、演示视频（可选）。
- **交付物**：讨论区与教师模块、全量 API 文档、用户手册、测试/性能报告、项目 PPT、演示视频（可选）。

## 5. 功能需求（按模块）
### 5.1 用户与认证
- 邮箱/学号注册、登录、密码重置（迭代 1 完成登录）。
- 角色分配（学生/教师/管理员），JWT 鉴权，登录态刷新。
- 个人资料：头像上传、资料修改、密码修改。
- 用户状态启用/禁用、角色切换（管理员）。

### 5.2 课程与班级
- 课程基本信息管理（名称、学期、简介、封面）。
- 班级创建、学生加入、批量导入、成员管理。
- 课程公告、资料上传（迭代 2）。
- 课程进度/模块管理（可选，迭代 2）。

### 5.3 作业管理
- 作业发布：题目/描述、文件附件、截止时间、评分标准、可重复提交策略。
- 提交方式：在线答题 + 文件上传；支持查看提交状态、历史记录。
- 批改：客观题自动判分，主观题教师评分 + 评语；评分发布与撤销。
- 成绩计算：支持按作业类型配置权重（迭代 2）。

### 5.4 出勤管理
- 教师创建考勤任务（课程/日期/规则）；学生扫码或定位签到。
- 签到、签退、请假申请、异常考勤处理。
- 出勤统计：个人/班级视角，支持导出（迭代 2 完成报表）。

### 5.5 讨论区与互动（迭代 2 核心）
- 讨论区：帖子列表、详情、发帖、编辑、删除。
- 评论：层级评论、@提醒、富文本基本能力。
- 公告发布：教师后台发布课程公告，学生可在首页看到。

### 5.6 通知与消息
- 事件驱动通知：作业截止提醒、成绩发布、考勤异常等。
- 消息中心：未读/已读状态、批量标记、站内信。
- 可选扩展：邮件/企业微信推送（迭代 2 评估）。

### 5.7 统计与 Dashboard
- 学生：作业完成度、出勤率、讨论活跃度。
- 教师：班级出勤/作业完成率、成绩分布。
- 管理员：系统活跃用户、API 调用、错误监控（可选）。

## 6. 非功能需求
| 维度 | 目标/要求 |
| --- | --- |
| 性能 | 常规操作响应 < 2s；支持 1000 并发；大列表分页 |
| 安全 | JWT + HTTPS（部署阶段）；密码加密、SQL 注入与 XSS 防护；操作日志 |
| 可用性 | 日志/监控；每日备份；出现故障 2h 内恢复 |
| 兼容性 | Chrome、Edge、Firefox 最新版；1280px+ 响应式；移动端核心功能（签到、查看作业） |
| 可维护性 | 代码风格校验、单元测试覆盖率 ≥ 50%（后端核心模块） |

## 7. 数据与集成
- **数据类型**：用户、课程、班级、作业、提交、出勤、讨论、通知、文件、统计。
- **外部集成**：邮件服务（SMTP）、对象存储（可选）、ApiFox/Swagger 作为接口协作工具。
- **数据治理**：每日增量备份，关键表（提交、出勤）具备软删除/版本字段。

## 8. 接口与文档
- 提供统一 RESTful API，遵循 `/api/v1` 前缀。
- 统一响应结构：`code/message/data/timestamp`。
- 提供 ApiFox 项目、Swagger UI、OpenAPI（`API.json`）文档，随版本迭代更新。
